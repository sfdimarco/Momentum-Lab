# Project Constraints & Synesthesia Grammar

## Strict Operational Constraints
1. **Zero Additive Logic**: Do not suggest new features, new modules, or external crates/libraries unless explicitly asked. The objective is to refine and debug the exact logic provided.
2. **Respect the Core Mapping**: The system relies on a literal, unchangeable mapping system. Do not abstract, metadata-ize, or alter base arrays. Treat mappings as hard physical laws.
3. **Strict Memory & Boundary Focus**: When reviewing Rust/Wasm, focus exclusively on memory safety, borrowing, and the efficiency of the WebAssembly bridge.
4. **No Extraneous Code**: When fixing a function, output only that fixed function. Do not rewrite surrounding architecture.
